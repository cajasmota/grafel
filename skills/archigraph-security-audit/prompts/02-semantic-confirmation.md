# Phase 2 — LLM semantic confirmation

Reads `phase1-findings.json`, confirms or dismisses each finding with LLM reasoning, and persists confirmed findings to the graph and doc store.

## Pre-flight

- Load `phase1-findings.json` from `~/.archigraph/groups/<group>/archigraph-security-audit/`.
- Load prior `~/.archigraph/groups/<group>/archigraph-security-audit/state.json` (if exists) to skip already-confirmed findings by fingerprint.
- If no findings remain after deduplication: print "No new findings to confirm." and exit.

## Step 1 — Cost estimate

Print before any LLM calls:
> Phase 2: **N** findings to review (~X tokens). Proceed? [y/N]

Gate on user confirmation unless `--auto` is set.

## Step 2 — Process findings

For each finding in priority order (critical first, then high, medium, low, info):

1. Call `archigraph_expand(node=<entity_id>, depth=2)` to get neighbour context.
2. If tech docs exist at `~/.archigraph/docs/<group>/<repo>/modules/`, load the relevant module README section.
3. Construct an LLM prompt:
   > Entity: `<name>` (`<kind>`) in `<repo>`
   > Static finding: `<check_name>` — `<summary>`
   > Graph context: `<expand output>`
   > [Optional] Module context: `<README excerpt>`
   >
   > Is this a real security issue or a false positive? Assign severity (critical/high/medium/low/info) and write a 2-4 sentence explanation a developer can act on.

4. Parse LLM response: extract `severity`, `is_false_positive` (bool), `explanation`.

5. **In interactive mode** (default): present to user:
   > **[<SEVERITY>]** `<entity_name>`: `<explanation>`
   > Confirm? (y)es / (n)o dismiss / (e)dit severity / (s)kip

6. **In `--auto` mode**: auto-confirm all findings with `is_false_positive=false` and `confidence >= 0.7`.

## Step 3 — Persist confirmed findings

For each confirmed finding, call:
```
archigraph_save_finding(
  type="security_finding",
  question="<check_name> on <entity_name>",
  answer="<explanation>",
  entity_id="<entity_id>",
  severity="<severity>"
)
```

Write a per-finding markdown page to `~/.archigraph/docs/<group>/security/findings/<fingerprint>.md`:

```markdown
# <check_name>: `<entity_name>`

**Severity:** <SEVERITY>
**Entity:** `<entity_name>` (`<kind>`) — `<repo>`
**Check:** `<check_name>`

## Finding

<explanation>

## Static evidence

<summary from phase1-findings.json>

## Remediation

<1-3 bullet points the developer should act on>

---
*Generated by archigraph-security-audit Phase 2. Fingerprint: `<fingerprint>`.*
```

## Step 4 — Write index

After all findings are processed, write `~/.archigraph/docs/<group>/security/index.md`:

```markdown
# Security audit — `<group>`

**Run date:** <date>
**Mode:** <interactive|auto>

## Summary

| Severity | Count |
|----------|------:|
| Critical | C |
| High | H |
| Medium | M |
| Low | L |
| Info | I |
| Dismissed | D |

## Findings

<table with one row per confirmed finding: severity, entity name, check name, link to finding page>

---
*Run `/archigraph-security-audit` again with `--since <sha>` to check only changed entities.*
```

## Step 5 — Update state

Write `~/.archigraph/groups/<group>/archigraph-security-audit/state.json`:
```json
{
  "last_run_timestamp": "<rfc3339>",
  "last_run_sha": "<sha or null>",
  "confirmed_fingerprints": ["<fp1>", "<fp2>"],
  "dismissed_fingerprints": ["<fp3>"]
}
```

## Reporting

Print:
> Phase 2 complete: **N** confirmed, **D** dismissed, **S** skipped. Security docs written to `~/.archigraph/docs/<group>/security/`.
